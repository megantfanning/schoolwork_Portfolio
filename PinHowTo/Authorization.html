Authorization

Pinterest is a RESTful API uses OAuth 2.0 for authentication. The Pinterest API uses OAUTH and allows both read and write permissions when interacting with a user's content. The process of gaining authorization to act of a users behalf is several steps. First we request permission from the user, then we receive an authorization code which we can exchange for a token to make changes.
You’ll make requests using your app ID and app secret. These let us know that it’s actually you making the request. Since your app secret gives you access to your users' profiles, boards and Pins, treat it with care and don't share it publicly. All requests must be made over https. We will not redirect your requests from http to https.

Authorization with API

Create a access token If you have problems with your token Pinterest has a token debugger tool After you get your access code, you can request an access token. Make a POST request to https://api.pinterest.com/v1/oauth/token with the following parameters.
grant_type Must take the value authorization_code.
client_id : Your app ID. You can get this ID from your app page.
client_secret : Your app secret. You can get this from your app page.
code : The access code you received from your redirect URI.

Authorization with Javascript SDK

When setting up your SDK you’ll need to add redirect URIs to use the SDK. You can add redirect URIs in the Platform: Web section of your app page. Remember that OAuth only accepts https connections. Redirects are used as fallbacks if a browser doesn't support the postMessage API. You will use the authentication methods to log users into your program (.login, .logout,.getsession,.setsession).
