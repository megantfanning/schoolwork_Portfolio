Pinterest API Endpoints

Pinterest uses a restful API. A restful API is a API which returns internet media in response to queries, typically JSON strings (which are human and machine readable), and uses standard HTTP methods to request and alter the data (in this case GET).You make a request to a URL, just like going to a site in a web browser, but instead of returning HTML, it returns JSON. This is an example of what a API request might look like is
https://api.pinterest.com/v3/domains//pins/top/clicks/
to make a request with a access token you need to send a request like this
https://api.pinterest.com/v3/users/myUserName/?access_token=APIKEY

To see what a response might look like look at this.
All calls have a limited return of 1,000 pins unless otherwise specified. When querying the server for information you can choose a period of time to look at that data for, in Pinterest the options are daily, weekly, or bi-weekly. This is called the aggregation value which must be provided in a string can be one of the following values:biweekly, daily, monthly, or weekly. The end_date is the parameter which sets the end point for aggregation. The end date should be provided in A date string in the format YYYY-mm-dd.
You can explore the API using their Explorer Tool
