The Pinterest API is leverage by the Pinterest SDK. There are two categories of methods in the SDK, authentication and users, boards, and Pins.
Authentication handles logging in a out and of each session as well as getting and setting sessions. Users boards and pins are requested using the .me (users), .pin(does the pinning action)
You can use PDK.request for all API endpoints not covered by other convenience methods. See the above list of links for users,  boards and Pins for a list of all available endpoints. All PDK.request calls require an authenticated user
The SDK will let you know if there is more information by giving a hasNext response. Use the response.next() helper function to fetch the next page of results.

var pins = [];
PDK.request('/boards//pins/', function (response) { // Make sure to change the board_id
if (!response || response.error) {
alert('Error occurred');
} else {
pins = pins.concat(response.data);
if (response.hasNext) {
response.next(); // this will recursively go to this same callback
} } });
For more information on specific calls see the Pinterest documentation
